# 类型运算与工具类型

## 类型运算

### keyof

`keyof`运算符的操作对象始终是类型，它的使用有两种情况。

1. 生成对象类型的key的联合类型。

   ```typescript
   type point = {
       x: string,
       y: string,
   }
   
   type p = keyof point;  // 'x' | 'y'
   ```

2. 如果类型具有索引签名，则返回索引的类型。

   ```typescript
   type arrish = { [k: string]: string};
   type a = keyof arrish;  //  string
   
   type mapish = { [k: number]: number };
   type m = keyof mapish; // string | number  因为javascript对象的键始终被强制转化为字符串，所以这里是联合类型
   ```

### typeof

`typeof` 操作符其实比较好理解，在元素JavaScript就有这个操作符，用来返回变量的类型。如果在TypeScript直接使用的话是没有什么意义的，因此需要结合其他工具使用。

```typescript
function count(a: number, b: number): number {
    return a + b;
}

type returnType = ReturnType<typeof count>;
```

### 索引访问类型

索引类型根据操作对象的不同，有3种情况。

1. 操作对象是对象类型。

   ```typescript
   type person = {
       name: string,
       age:  number,
       gender: boolean, 
   };
   
   type a = person['name']; // string
   type b = person['name' |'age']; // string | number
   type c = person[keyof person]; // string | number | boolean
   ```

2.  操作对象是对象数组，此时需要结合`typeof`操作符使用，此时会返回数组中每个元素类型的联合类型。

   当数组元素存储的是对象类型的值的时候。

   ```typescript
   const users = [{
       name: 'John',
       age:  19, 
   }, {
       name: 'Doe',
       sex: true
   }];
   
   
   type user = typeof users[number];
   // 此时user的类型如下：
   type user = {
       name: string;
       age: number;
       sex?: undefined;
   } | {
       name: string;
       sex: boolean;
       age?: undefined;
   }
   ```

   当数据存储的的基本类型的值的时候也是一样的。

   ```typescript
   const arr = ['submit', 'button', 3] ;
   
   type a = typeof arr[number];
   // 此时a表示的类型如下：
   type a = string | number
   ```

3. 当操作的对象是数组，并且数组存储的是基本类型的数据的时候，结合`as const`使用的时候，表示的是数组成员字面量的联合类型。

   ```typescript
   const arr = ['submit', 'button', 3] as const;
   type a = typeof arr[number];
   // 此时a标识的类型如下:
   type a = 'submit' | 'button' | 3;
   ```

### 类型断言

类型断言指的是，它允许我们手动设置变量的数据类型并防止编译器自行推断它。

类型断言的使用有两种方式：

1. 使用`as`关键字。

   ```typescript
   const myCanvas = document.getElementById("main_canvas") as HTMLCanvasElement;
   ```

2. 使用尖括号。

   ```typescript
   const myCanvas = <HTMLCanvasElement>document.getElementById("main_canvas");
   ```

TS 只允许将类型断言转换为更具体或更不具体的类型。这个规则防止了“不可能”的强制转换，比如：

```typescript
const x = "hello" as number;
```

类型断言的缺点

类型断言的使用实际上也是有一个缺点的

1. 它可能导致运行时错误：当使用“as”执行类型断言时，编译器被迫接受开发人员的断言为真，即使事实并非如此。如果类型断言不正确，这可能会导致运行时错误。
2. 它可以禁用类型检查：当使用“as”执行类型断言时，编译器对该变量或表达式的类型检查被禁用。这会使捕获类型错误变得更加困难，并可能导致代码中出现更多错误。
3. 它可能会降低代码的可读性：过度使用“as”关键字可能会使代码更难以阅读和理解，特别是对于不熟悉代码库的开发人员而言。
4. 这可能是设计不佳的标志：在某些情况下，需要使用“as”进行类型断言可能表明代码设计中存在缺陷，例如类型信息不足或接口定义不明确。



## 工具类型

工具类型像是类型系统的“转换器”，可以理解为操作类型的“函数”。

### 属性修饰类

这类工具不改变属性的数量，只改变它们的可选性或读写性。

|工具类型     |作用	                | 核心原理简述            |
| ------------ | ------------------- | ------------------------- |
|`Partial<T>`	 |将所有属性变为可选 (?) |遍历所有 Key，加上 ?        |
|`Required<T>` |将所有属性变为必选     |遍历所有 Key，去掉 -?     | 
|`Readonly<T>` |将所有属性变为只读	 |遍历所有 Key，加上 readonly |


### 结构挑选类

这类工具会从一个复杂的对象类型中，精准地“切下”一部分。

- `Pick<T, K> `
    
    从类型 `T` 中挑选出属性 `K` 中的属性。
    ```typescript
    interface Todo { title: string; desc: string; done: boolean; }
    type TodoPreview = Pick<Todo, "title" | "done">; // 只要这两个属性
    ```

- `Omit<T, K>` 
    
    从类型 `T` 中排除属性 `K` 中的属性。
    ```typescript
    interface Todo { title: string; desc: string; done: boolean; }
    type TodoPreview = Omit<Todo, "desc">; // 排除desc属性
    ```

### 集合处理类

这类工具通常用于从联合类型中过滤掉不需要的部分。

- `Exclude<T, U>`

    从 `T` 中排除掉可以赋值给 `U` 的类型。
    ```typescript
    type T = Exclude<"a" | "b" | "c", "a">; // 结果: "b" | "c"
    ```

- `Extract<T, U>`

    从 `T` 中提取出可以赋值给 `U` 的类型（求交集）。
    ```typescript
    type T = Extract<"a" | "b" | "c", "a" | "f">; // 结果: "a"
    ```
    
- `NonNullable<T>`

    从 `T` 中排除掉 `null` 和 `undefined` 类型。
    ```typescript
    type T = NonNullable<"a" | "b" | "c">; // 结果: "a" | "b" | "c"
    ```

### 映射与函数类

这类工具处理的是更抽象的逻辑。

- `Record<K, T>`
    
    创建一个对象类型，其键为 `K`，值为 `T`。这是定义配置对象、映射表的利器。
    - `K` (Keys)：你想给对象定义的键的集合（通常是一个联合类型，比如 `"a" | "b" | "c"`）。
    - `T` (Type)：你想给这些键统一定义的值的类型（比如 `string`、`number` 或`interface`）。

    示例1：当你有一组固定的页面，每个页面需要对应的权限配置：
    ```typescript
    type Page = "Home" | "Admin" | "Settings";

    interface PageConfig {
    title: string;
    isPublic: boolean;
    }

    const navConfig: Record<Page, PageConfig> = {
    Home: { title: "首页", isPublic: true },
    Admin: { title: "管理页", isPublic: false },
    Settings: { title: "设置页", isPublic: false }
    };
    ```

    示例2：
    这是最常见的用法之一。当你有一个枚举时，用 Record 确保你处理了枚举里的每一个成员。
    ```typescript
    enum UserRole {
        Admin,
        Guest
    }

    const roleLabel: Record<UserRole, string> = {
        [UserRole.Admin]: "管理员",
        [UserRole.Guest]: "访客"
    };
    ```

- `ReturnType<T>`

    获取函数类型的返回值类型。这在处理第三方库或复杂的工厂函数时极其有用。

    示例1：
    ```typescript
    type T = ReturnType<typeof setTimeout>; // 结果: number
    ```

    示例2:
    ```typescript
    function f() { return { x: 10, y: 20 }; }
    type P = ReturnType<typeof f>; // 结果: { x: number, y: number }
    ```
