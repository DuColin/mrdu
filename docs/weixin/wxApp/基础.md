# 基础


小程序的技术框架深度借鉴了 Vue 的模板语法与 React 的组件化思想。

## 代码构成

在小程序官网中，代码构成包含 WXML、 WXSS、 JS 和  JSON 文件，考虑到 JSON 只是做配置用，这里我们先不管它。

虽然它们看起来和网页开发的 HTML、CSS、JS 很像，但在微信的“双线程架构”下，它们的底层逻辑和运行机制其实发生了根本性的变化。并且为了适配“微信安全管控”，在语法层面做了“增强”和“限制”。

## WXML

| HTML 标签 | WXML 对应组件| 说明 |
|-----------|-------------------------| ----- |
|`<div>`    |`<view>`                 | 块级容器 |
| `<span>`  |`<text>`                 |行内文本（只有 `<text>` 支持长按选中）|
| `<img>`   |`<image>`                |图片（必须指定宽高，支持懒加载）|
|`<a>`      |`<navigator>`            |页面跳转（不能跳外链）|
|`<ul>` / `<li>` | `<view>` + `wx:for`| 列表（无原生列表标签）|
|`<input>`  | `<input>` / `<textarea>`|	表单（样式受限，需用 placeholder-style）|


### 数据绑定

使用双花括号 `{{}}` 将 JS 中的数据动态渲染到视图上。

- 基础绑定

    ```xml
    <view>{{ message }}</view>
    <image src="{{ imageUrl}}"><image>
    ```

    ```js
    Page({
        data: {
            message: 'Hello World'
            imageUrl: '/images/logo.png'
        }
    })
    ```

- 支持简单运算， 如三元运算、加减乘除、对象或者数组的访问。

    ```xml
    <view>
        <text>{{ a + b }}</text>
        <text>{{ user.name }}</text>
        <text>{{ list[0].name }}</text>
    </view>
    ```

    ```js
    Page({
        data: {
            a: 2, 
            b: 3,
            user: {
                name: '张三'
            },
            list: [
                {
                    name: 'Apple',
                    age: 13
                }
            ]
        }
    })
    ```


### 列表渲染

用于循环渲染数组，类似于 Vue 的 `v-for`。

- 默认使用 `item` 和 `index` 作为当前项和索引的变量名。
- 可以使用 `wx:for-item` 和 `wx:for-index` 来指定当前项和索引。
- 可以使用 `wx:key` 为当前项指定 `key`。用于优化列表的渲染性能，确保列表项的顺序变化时能高效更新。
    如果是数字或者字符串数组，可以使用 `this` 关键字代表 `item` 本身; 如果是纯静态列表可以使用 `index`；其他情况则
    必须指定列表项中的属性，属性值唯一且为字符串或数字值

```xml
<view>
    <!-- 默认使用 item 作为当前项， index 作为索引 -->
    <view wx:for="list">{{ index }} . {{ item.title }}</view>

    <!-- 使用 wx:for-item 指定当前项，使用wx:for-index 指定索引 -->
    <view wx:for="list" wx:for-item="entry" wx:for-index="no">
        {{ no }} . {{ entry.title }}
    </view>

    <view wx:for="list" wx:key="index">{{ index }} . {{ item.title }}</view>

</view>
```

```javascript
Page({
    data: {
        list: [
            { title: '标题一' }
            { title: '标题二' }
            { title: '标题三' }
            { title: '标题四' }
            { title: '标题五' }           
        ]
    }
})
```

### 数据监听器


- 监听整个对象。

    可以使用 `**` 通配符监听对象的所有子数据的变化。

    ```javascript
        Page({
            data: {
                user: {
                    name: '',
                    detail: {
                        province: ''，
                        city: '',
                    }
                }
            }
            observers： {
                // 触发条件：
                // #1 使用 setData 设置 this.data.user.detail 本身或其下任何
                //    子数据字段时触发
                // #2 使用 setData 设置 this.data.user 也会触发
                'user.detail.**': (newVal) => {
                    // 回调参数：
                    // newVal 接收到的是整个 user.detail 对象的最新值，而不是
                    //        变化的那个字段。
                    console.log('数据变化'， newVal)
                } 
            }
            attached: () => {
                // 这样会触发上面的 observer
                this.setData({
                    'user.detail': { /* ... */ }
                });

                // 这样会触发上面的 observer
                this.setData({
                    'user.detail.city': 'xxx'
                });

                // 这样也会触发上面的 observer
                this.setData{
                    user: {/* ... */}
                }
            }
        })
        ```
- 监听多个字段或属性

    可以用逗号 `,` 分隔，同时监听多个属性。

    ```javascript
    Page({
        data: {
            a: 0, 
            b: 0,
            sum: 0,

            user: {
                name: '张三',
                age: 19,
                detail: {
                    province: '陕西',
                    city: '西安'
                }
            }
        },
        observers: {
            // #obs1
            'a, b': (a, b) => {
                this.setData({
                    sum: a + b
                })
            },
            // #obs2
            'user.name, user.age': (newName, newAge) => {
                console.log('名字或年龄变了', newName, newAge);
            }
        },
        attached: () => {

            // 触发obs1
            this.setData: {{
                'a': 3
            }};

            // 触发obs1
            this.setData: {{
                'b': 4
            }};

            // 触发obs2
            this.setData({
                'user.name': 'Blob'
            });

        }
    })
    ```

- 监听数组

    某些场景下，我们可能需要监听数组的变化。

    - 监听整个数组的变化。
    
        监听的数组是数组本身（即引用）的变化，内部某一项属性的变化是不会被监听到的。
        可以监听到数组长度的变化。

    - 监听数组及内部属性的变化。

        和对象的监听一样，使用 `**` 通配符去监听。但只能监听到数组“内部元素”的变化，很难监听数组本身的变化，如“长度”。

    - 监听数组的某一项。

    - 监听数组内对象的属性。

    ```javascript
    Page({
        data: {
            // 简单数组
            tags: ['前端', '后端'],

            // 对象数组
            list: [
                { name: 'Apple', price: 5 },
                { name: 'Banana', price: 3 }
            ]
        },
        observers: {
            // #obs1 监听整个 list 数组的变化
            'list': function(newList) {
                console.log('可以监听到数组长度变化');
                console.log('数组整体被替换了:', newList);
            },
            // #obs2 监听 list 数组及内部属性的变化
            'list.**': function(newList) {
                console.log('无法监听到数组长度变化');
                console.log('数组内部属性发生变化',  newList)
            },
            // #obs2 监听第 0 个用户的 name 属性
            'list[0].name': function(newName) {
                console.log('第一个用户改名了:', newName);
            }
        }
    })
    
    ```
