# 开发框架：小程序全览

## 小程序创建

微信小程序的创建实际上包括账号注册、项目搭建、发布上线的闭环。[账号注册](https://developers.weixin.qq.com/miniprogram/dev/framework/quickstart/getstart.html#%E7%94%B3%E8%AF%B7%E8%B4%A6%E5%8F%B7)参考官方文档。核心区别在**项目搭建**环节，这里我们介绍三种主流的创建方式：

1. 原生开发
2. 跨平台框架开发
    
    基于 Taro、uni-app、WePY 等第三方框架开发，最终编译为小程序代码，支持多端同步发布。

3. 低代码开发

    通过可视化拖拽、模板套用的方式搭建，无需或极少编写代码，主流方式包括：
    - 微信官方低代码：微信开发者工具中集成的 微搭低代码，直接在工具内可视化搭建。
    - 第三方 SaaS 平台：如即速应用、有赞、微盟等，提供行业模板与拖拽编辑器。

    功能受模板限制、定制化能力弱、数据与平台绑定较深。



## 小程序项目结构

小程序项目目录结构通常有两种形式：扁平结构和包裹结构，他们的核心区别在于源码的存放位置。使用“微信开发者工具”创建项目时，选择不同的模板即可创建这两种结构的项目。

    │  ├─ index/
    │  │  ├─ index.js

下面我们介绍下，该结构下不同文件的作用。

### 项目配置文件

项目配置文件包括 `project.config.json` 和 `project.private.config.json` 这两个文件。
- 它们不参与小程序的运行，而是“微信开发者工具”所需的项目配置文件。
- 用于集中管理项目基本信息、开发环境设置和构建流程，确保开发一致性并提升团队协作效率。

具体的配置字段参考[文档](https://developers.weixin.qq.com/miniprogram/dev/devtools/projectconfig.html)


- `project.config.json` ：开发者工具专属配置， 包含AppID、项目名称、编译设置等
- `project.private.json` ： 这是个人私有配置文件
    - 用于覆盖 `project.config.json` 中的配置
    - 存储个人的调试偏好、自定义编译条件、本地路径
    - 不应该提交到 Git，避免污染团队配置

### 全局配置文件

这些是小程序的核心配置文件，作用于整个小程序。

##### app.js

它是小程序的入口文件，这个文件主要作用就是执行 `App()` 方法，也就是说 `App()`的作用就是这个文件的职责。

`App()` 是微信小程序框架提供的全局注册函数，它是整个小程序的入口和根对象。它的三个方面的职责
1. 注册小程序实例。每个小程序有且只有一个 `App()` 调用。
    
    ```js
    App({
        // ... 配置项
    })
    ```

2. 管理小程序的生命周期。

    ```js
    App({
        // 小程序启动触发
        onLunch() {
            console.log('小程序启动了......')
        },
        // 小程序从后台切换到前台触发
        onShow(options) {
            console.log('小程序切换到前台.......')
        },
        // 小程序从前台切到后台时触发
        onHidden() {
            console.log('小程序切换到后台.......')
        }
    })
    ```

3. 定义全局共享的数据和方法。
    
    有两种方式存储全局共享数据：
    
    1. 放在 `globalData` 内（推荐）。

        ```js
        App({
            globalData: {
                userInfo: null,
                token: ''
            }
        })
        ```
        
        使用方式：
        
        ```js
        const app = getApp();
        app.globalData.userInfo; // 读取
        app.globalData.token = 'xxx'; // 修改
        ```
    
    2. 放在 `globalData` 平级（可以，但不推荐）。
        
        ```js
        App({
            globalData: {},
            userInfo: null,
            token: null,
        })
        ```
        
        使用方式：
        ```js
        const app = getApp()
        app.userInfo; // 读取
        app.token = 'xxx'; // 修改
        ```

    全局方法定义方式：

    ```js
    App({
        globalData: {
            userInfo: null,
            token: '',
            isLogin: false,
        },
        // 设置用户信息
        setUserInfo(info) {
            this.globalData.userInfo = info;
        },
        // 设置 token
        setToken(token) {
            this.globalData.token = token;
            this.globalData.isLogin = !!token; 
        },
        checkLogin() {
            if (!this.globalData.isLogin) {
                wx.navigateTo({ url: '/pages/login/login' });
                return false;
            }
            return true;
        }
    })
    ```

    在页面中使用全局方法：
    
    ```js
    // pages/index/index.js
    const app = getApp();

    Page({
        data: {
            userName: '',
        },
        onLoad() {
            this.loadUserInfo()
        },
        loadUserInfo() {
            // 读取全局数据
            if (!app.globalData.isLogin) {
                this.setData({
                    userName: app.globalData.userInfo.name
                })
            }

            // ✅ 调用全局方法，简洁明了
            if (!app.checkLogin()) {
                return  // 未登录，已自动跳转
            }
        }

    })

    ```
    
   


 
##### `app.json`

[app.json](https://developers.weixin.qq.com/miniprogram/dev/reference/configuration/app.html)是小程序的全局配置文件，它大致可以配置以下内容：
- 小程序有哪些页面。
- 小程序窗口的样式和结构。
- 底部导航栏样式和行为。
- 网络行为控制，如超时时间等。

```json
{
  "pages": [
    "pages/index/index",
    "pages/logs/logs"
  ],
  "window": {
    "navigationBarTextStyle": "black",
    "navigationBarTitleText": "Weixin",
    "navigationBarBackgroundColor": "#ffffff"
  },
  "style": "v2",
  "componentFramework": "glass-easel",
  "sitemapLocation": "sitemap.json",
  "lazyCodeLoading": "requiredComponents"
}
```

上面是一个配置示例，解释下几个重要字段。
- `pages` ： 页面路由注册

    告诉微信框架这个小程序有哪些页面。
    - 第一个页面 = 首页（用户打开小程序时显示的第一个页面）。
    - 数组中的顺序就是页面的注册顺序。
    - 如果你在 `pages/` 目录下新建了页面，但没在这里注册，页面无法访问

-  `window` — 全局窗口外观配置
    
    设置所有页面顶部导航栏的默认样式。
    |属性	                        |含义	        |
    | ----------------------------- | ------------- |
    |navigationBarBackgroundColor 	|导航栏背景色	 |
    |navigationBarTextStyle	    	|导航栏文字颜色	 |
    |navigationBarTitleText	    	|导航栏标题文字	 |

- `componentFramework`

    指定使用的组件框架。

- `sitemapLocation`

   指定 `sitemap.json` 文件位置, 用于配置小程序的索引规则

##### `app.wxss` 

全局样式文件，作用于所有页面，但可被页面级样式覆盖

### 小程序组件框架

组件框架是小程序内部的“组件管理内核”，负责所有组件的底层运行逻辑。包括：

- 解析你写的 WXML 组件（`<view>` / `<text>` / 自定义组件）。
- 管理组件的创建、更新、销毁（生命周期）。
- 处理数据绑定 `{{}}`、属性传递、事件通信。
- 处理插槽 `slot`、列表渲染 `wx:for`。
- 做渲染优化（diff 对比、局部更新）。
- 连接小程序双线程架构（让逻辑层和渲染层的组件能通信）

对于有过 Vue 等框架经验的人来讲，这个过程并不陌生。但是实际上这两者之间并不一样。

小程序有两种框架：

|框架名称      |状态         |说明       |
| ----------- |----------- | ------------------------------------------------------------------- |
| exparser    | 旧版       | 小程序诞生以来一直使用的原始组件框架                                                   |
| glass-easel | 新版（推荐）| 2022年后推出的全新重写版本，对 exparser 的重写，性能更好，特性更多。对 `TypeScript`、`shandow-dom`有更好的支持|

#### 与 Vue、React 框架对比

与 Vue、React 等前端开发框架相比，小程序与他们存在一些相同点，但是也有不同。

它们做的事情几乎一样，只是"方言"不同。不管 Vue、React 还是小程序组件框架，它们都在解决同一个问题：组件化开发。

但是几个本质上的区别，决定了它们不是一类东西：

- 运行架构不同

    在[技术架构](./技术架构.md)这一章节我们讲了微信小程序运行架构和传统 web 架构的区别，

- 层级定位不同

    Vue 和 React 属于上层开发框架，是给开发人员使用的，

### 代码构成

在小程序官网中，代码构成包含 WXML、 WXSS、 JS 和  JSON 文件，考虑到 JSON 只是做配置用，这里我们先不管它。

虽然它们看起来和网页开发的 HTML、CSS、JS 很像，但在微信的“双线程架构”下，它们的底层逻辑和运行机制其实发生了根本性的变化。并且为了适配“微信安全管控”，在语法层面做了“增强”和“限制”。

#### WXML

##### 与 HTML 相比：标签体系完全不同
| HTML 标签 | WXML 对应组件| 说明 |
|-----------|-------------------------| ----- |
|`<div>`    |`<view>`                 | 块级容器 |
| `<span>`  |`<text>`                 |行内文本（只有 `<text>` 支持长按选中）|
| `<img>`   |`<image>`                |图片（必须指定宽高，支持懒加载）|
|`<a>`      |`<navigator>`            |页面跳转（不能跳外链）|
|`<ul>` / `<li>` | `<view>` + `wx:for`| 列表（无原生列表标签）|
|`<input>`  | `<input>` / `<textarea>`|	表单（样式受限，需用 placeholder-style）|
